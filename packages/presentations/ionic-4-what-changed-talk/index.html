Ionic 4

So, what's different?

tl;dr - Lots of good change, minimal bad change for users

lots of (private) implementation details changed

For starters

import { xyz } from 'ionic-angular'

 

import { xyz } from '@ionic/core'

import { abc } from '@ionic/angular'

Standards Based

Each UI control is no longer tightly coupled to Angular

Each control is implemented using the custom element browser spec

Custom Elements

So what does that mean?

The big one is directives are no longer first-class citizens in Ionic- menuClose is now <ion-menu-toggle>

Custom Elements

Ionic directives have been updated to either be attributes/properties, or a new DOM element for wrapping

Trivial upgrade for small apps, can be automated for large apps

Custom Elements

Angular tooling still plays nicely, including the language service

Ionic ships Angular specific stuff with `@ionic/angular` to make sure the Angular ecosystem plays nicely

Standards Based

Ionic uses CSS variables extensively for styling/theming

CSS variables are fricking awesome

Sass is now optional

Standards Based

Oftentimes, Ionic events had confusing names and were shared between multiple components

Not only was naming cleaned up, but Ionic now uses standard DOM events

Bundle Size

The biggest complaint we heard about Ionic 2/3 was how long it took for apps to boot

There is a direct correlation between boot time and the outputted bundle size

Bundle Size

Ionic tackled this head on and made it a core tenant of Ionic 4

It's complicated, but the gist is only components that are in the DOM are loaded

This made Ionic 4, in it's purest sense, super fast

Cue "do it live" explanation of how this works :-D

tl;dr - Ionic 4 is really fast and lightweight compared to previous versions

Developer Experience

Finally, a complete SDLC story

Ionic 4 CLI uses Angular CLI under the hood

This means it's super fast

But more importantly, first-class testing

Unit tests

E2E tests

Environment support, etc

Ionic tooling matches the hight and tight Angular developer experience

Dependency Injection

Ionic internally doesn't use DI anymore

Ionic still works with DI for your apps

Some changes in the provider APIs, since providers are loaded asynchronously now
